# /task-list — プランからタスクリストを生成

`/plan` で作成した計画をタスクリスト（TaskCreate）に変換し、実装中の進捗を追跡可能にする。
オートコンパクトが発生してもタスク状態が保持されるため、長い実装セッションで特に有効。

## 使い方

```
/task-list              # 現在のプランファイルからタスクリストを生成
/task-list status       # 現在のタスク状態を表示
/task-list done <番号>   # 指定タスクを完了にする
```

## 引数判定

スキル起動時、引数（args）を確認して分岐する:

- **引数なし** → 「フロー」セクションを実行（タスク生成）
- **`status`** → `TaskList` を呼び出してサマリー表を表示。タスク生成は行わない
- **`done N`** → N番目のタスクを `TaskUpdate` で `completed` に変更。依存チェックなし（先行タスクが未完了でも完了にできる）

## フロー

### 0. 重複チェック

まず `TaskList` を呼び出し、既存タスクがあるか確認する。
同名のタスクが既に存在する場合はスキップし、新規タスクのみ追加する。
全タスクが既に存在する場合は「タスクリストは作成済みです」と表示して終了。

### 1. プランファイルの検出

以下の順でプランファイルを探す:

```bash
# 1. プロジェクト別プランディレクトリ（system-reminderにplanファイルパスが表示される場合はそれを使う）
# 2. グローバルプランディレクトリ
ls -t ~/.claude/plans/*.md 2>/dev/null | head -1
```

会話中の `system-reminder` にプランファイルのパスが表示されている場合は、そのパスを優先的に使用する。
プランファイルが見つからない場合はエラーメッセージを表示して終了。

### 2. タスクの抽出

プランファイルから `## 数字.` または `## Step 数字` パターンのセクションを抽出する。
各セクションの見出しがタスク名、本文がタスクの説明になる。

抽出ルール:
- `## 1. DB マイグレーション` → タスク名: `1. DB マイグレーション`
- `## 2. 型定義` → タスク名: `2. 型定義`
- 番号付きセクションがない場合は `## 変更ファイル` のテーブルや箇条書きから抽出を試みる
- それでもなければプラン全体を1タスクとして登録

### 3. TaskCreate で登録

各タスクについて TaskCreate を実行:

```
TaskCreate:
  subject: "1. DB マイグレーション"
  description: "(セクション本文の要約 — 変更ファイル、主要な作業内容)"
  status: "pending"
```

**依存関係の設定**:
- デフォルトは**全直列**（各タスクは直前のタスクに `addBlockedBy` を設定）
- プランのコンテキストから明らかに並行可能なタスクのみ依存を外す（例: 独立したコンポーネント同士）
- 判断に迷う場合は直列にする（安全側）

### 4. サマリー表示

登録完了後、以下の形式でユーザーに報告:

```
━━━ タスクリスト作成完了 ━━━

プラン: {プランタイトル}
タスク数: {N}
タスクリストID: {CLAUDE_CODE_TASK_LIST_ID}

| # | タスク | 状態 | 依存 |
|---|--------|------|------|
| 1 | DB マイグレーション | pending | — |
| 2 | 型定義 | pending | #1 |
| 3 | サービス関数 | pending | #2 |
| 4 | コンポーネント | pending | #3 |
| 5 | 統合 | pending | #4 |

次のセッションで引き継ぐには:
  export CLAUDE_CODE_TASK_LIST_ID="{id}"

━━━━━━━━━━━━━━━━━━━
```

## 実装中の運用

### タスク着手時
作業開始時に `TaskUpdate` で `status: "in_progress"` に変更。

### タスク完了時
作業完了時に `TaskUpdate` で `status: "completed"` に変更。
コミット＆PR作成後が目安。

### 状態確認
`/task-list status` で `TaskList` を呼び出し、現在の進捗を表示。

### 手動完了
`/task-list done 3` で指定番号のタスクを completed に変更。
依存関係のチェックは行わない（先行タスクが未完了でも完了にできる）。

## 注意事項

- **プランなしで使うな**: 必ず `/plan` でプランを作ってからこのスキルを使う
- **タスクの粒度はプランに従う**: プランのセクション = タスク。勝手に分割・統合しない
- **オートコンパクト対策**: タスクリストはツール内部に保持されるため、会話が圧縮されても残る
- **セッション跨ぎ**: タスクリスト作成時に表示される ID を `CLAUDE_CODE_TASK_LIST_ID` 環境変数に設定すると、次のセッションでも同じタスクリストを参照できる
